What is real, what is constrained, and what is simulated

Pipeline

How the source hierarchy feeds the dashboard

The goal is a clean chain from public benchmarks to executive decisions.

01

Pull benchmark inputs

Use IPEDS and College Scorecard to define the class-size, yield, tuition, and net-price envelope.

02

Apply admissions guardrails

Use Common Data Set ranges to shape academic profile assumptions and selectivity logic.

03

Set benchmark guardrails

Use those source ranges as guardrails for class size, selectivity, pricing, and academic profile assumptions.

04

Generate synthetic students

Create student-level funnel, aid, deposit, and melt records within those constraints.

05

Validate outcomes

Check that rates, volumes, pricing, and mix stay inside the intended benchmark envelope.

06

Drive executive decisions

Use the constrained synthetic layer to answer class-target, conversion, aid, and melt questions.

Where the information comes from and what it is used for

This is the public-facing logic layer: source, use case, and why each source matters.

Source What We Take From It What We Do With It
